What is EiPL?

You see code. You write what it does — not how, but what.

EiPL (Explain in Plain Language) flips traditional coding exercises. Instead of writing code, you describe existing code in your own words. An AI then tries to recreate the code from your description alone.

Reverse engineering: prove you understand by describing.

1. Reference Code
2. Your Description
3. AI Generates Code
4. Tests Verify

PARTIAL Code

"This program asks for a number and prints a right-angled triangle of asterisks with that many rows."

Terminal Output
$ python greet.py
Name: Priya
Age: 20
Hello Priya!
You will be 30 in 10 years.
Good Description

"This program reads a name and age, then calculates how many years until the user turns 30."

COMPLETE
Terminal Output
$ python check.py
Enter text: racecar
racecar is a palindrome!

$ python check.py
Enter text: hello
hello is not a palindrome.
Bad Description

"This program reverses a string and prints it."

INCOMPLETE

The program checks for palindromes — it doesn't just reverse.

Prompt problems test inference from I/O examples. Study all the runs shown, not just the first one — edge cases reveal the true operation.

Tips for Success

🎯
Describe WHAT, not HOW
Focus on purpose and behavior. "Returns the largest element" beats "iterates through the list comparing each element."
✅
Correctness First
Pass all tests before worrying about abstraction level. The comprehension check only runs after 100% correctness.
💬
Use Your Own Words
A clearer, more natural description produces better AI-generated code. Write as if explaining to a friend.
